Database Location

macOS/Linux:

~/Library/Application Support/diigo-tagger/tags.db

Related files: tags.db-shm (shared memory), tags.db-wal (write-ahead log)

✓ Backup Operations (SAFE)

Always backup before dangerous operations!

Create Backup

cp -r "$HOME/Library/Application Support/diigo-tagger" \
   "$HOME/Library/Application Support/diigo-tagger.backup-$(date +%Y%m%d-%H%M%S)"

Restore from Backup

  1. Stop the application first!
  2. Remove current database: rm -rf "$HOME/Library/Application Support/diigo-tagger"
  3. Restore: cp -r "$HOME/Library/Application Support/diigo-tagger.backup-YYYYMMDD-HHMMSS" ...

Drop and Recreate Database

🚨 NUCLEAR OPTION - READ THIS FIRST 🚨

⚠️ THIS WILL PERMANENTLY DELETE:

  • All bookmarks in the local database
  • All tags and tag associations
  • All user edits and modifications
  • CANNOT BE UNDONE WITHOUT BACKUP

✓ This will NOT delete:

  • Bookmarks on Diigo.com (they're safe)
  • Your Diigo account data

When to use this:

  • You want to start fresh with a clean database
  • Your database has corrupted date fields from old sync code
  • You've verified backups exist and you're ready to lose local data

Procedure

Step 1: BACKUP (MANDATORY)

cp -r "$HOME/Library/Application Support/diigo-tagger" \
   "$HOME/Library/Application Support/diigo-tagger.backup-$(date +%Y%m%d-%H%M%S)"

Step 2: Stop Application

Press Ctrl+C in terminal running uvicorn

Step 3: Drop Database

rm -f "$HOME/Library/Application Support/diigo-tagger/tags.db"
rm -f "$HOME/Library/Application Support/diigo-tagger/tags.db-shm"
rm -f "$HOME/Library/Application Support/diigo-tagger/tags.db-wal"

Step 4: Recreate Database

cd /path/to/diigo_tagger
poetry run python -c "from diigo_tagger.db import init_db; init_db(); print('Database initialized')"

Step 5: Full Sync

Start server and go to:

Go to Sync Page

Select "Full Sync" and wait 15-30 minutes for completion

Migration Scripts

Migration scripts fix data issues without dropping the database.

Date Migration

Copies diigo_created_at → created_at for bookmarks synced with old code. 

cd /path/to/diigo_tagger
python scripts/migrate_created_dates.py

✓ Safe to run - only updates dates, doesn't delete data. But backup first!

Manual SQL Queries

✓ Read-Only Queries (SAFE)

Database Stats 
sqlite3 "$HOME/Library/Application Support/diigo-tagger/tags.db" "
SELECT 'Bookmarks:', COUNT(*) FROM bookmarks
UNION ALL
SELECT 'Tags:', COUNT(*) FROM tags
UNION ALL
SELECT 'Tag Associations:', COUNT(*) FROM bookmark_tags;
"
Date Distribution by Year 
sqlite3 "$HOME/Library/Application Support/diigo-tagger/tags.db" "
SELECT
    strftime('%Y', created_at) as year,
    COUNT(*) as count
FROM bookmarks
GROUP BY year
ORDER BY year;
"
Most Popular Tags (Top 20) 
sqlite3 "$HOME/Library/Application Support/diigo-tagger/tags.db" "
SELECT
    t.name,
    COUNT(bt.bookmark_id) as usage_count
FROM tags t
LEFT JOIN bookmark_tags bt ON t.id = bt.tag_id
GROUP BY t.name
ORDER BY usage_count DESC
LIMIT 20;
"

⚠️ Destructive Queries (DANGEROUS)

These modify or delete data. BACKUP FIRST!

Delete Orphaned Tags

Find them first (safe):

sqlite3 ... "SELECT id, name FROM tags t WHERE NOT EXISTS (...)"

Then delete (DESTRUCTIVE):

sqlite3 ... "DELETE FROM tags WHERE id IN (...)"
Vacuum Database (Reclaim Space) 
sqlite3 "$HOME/Library/Application Support/diigo-tagger/tags.db" "VACUUM;"

Locks database during operation but is safe

Recovery Procedures

Database Corruption

Symptoms: "database disk image is malformed", queries hang, missing data

  1. Stop application
  2. Create backup (even corrupted DB might be partially recoverable)
  3. Try SQLite recovery: dump → recreate → restore
  4. If recovery fails: drop and recreate

Accidental Deletion

If database still open: DON'T close it! Immediately restore from backup.

If already closed: Restore from backup or drop + re-sync from Diigo

Display IDs

What is a Display ID?

A Display ID is a short, 8-character identifier for each bookmark, making it easy to reference bookmarks without typing long URLs.

Example: 667ac2eb

How is it Generated?

Display IDs are generated using a CRC32 hash of the bookmark's URL, then converted to hexadecimal format.

  • Deterministic: Same URL always produces the same Display ID
  • Collision-resistant: Different URLs extremely unlikely to produce the same ID
  • Short and memorable: Only 8 characters

Where Can You Use Display IDs?

1. CLI Commands

Look up a specific bookmark:

diigo lookup 667ac2eb

Look up multiple bookmarks at once:

diigo lookup 667ac2eb 26793dba 002e22ba

2. API Endpoints

Fetch bookmark details via REST API:

GET /api/v1/bookmarks/667ac2eb

3. Search Queries

Reference bookmarks in documentation or notes:

See bookmark 667ac2eb for swimming tutorial details

Benefits

  • Easy to type: Much shorter than full URLs
  • Human-friendly: Hexadecimal characters only (0-9, a-f)
  • Unique per URL: Each bookmark has a unique identifier
  • Persistent: ID doesn't change as long as the URL stays the same
  • Copy-paste friendly: 8 characters fit easily in messages, notes, and scripts

Technical Details

For developers and curious users: 

import zlib

def generate_display_id(url: str) -> str:
    """Generate 8-character hex ID from URL using CRC32."""
    crc = zlib.crc32(url.encode('utf-8')) & 0xffffffff
    return format(crc, '08x')

# Example:
generate_display_id("https://www.youtube.com/watch?v=wrHL1igBhOg")
# Returns: "667ac2eb"
